# 发布指南

## 方式一：发布到 Chrome 应用商店（推荐）

这是最正规的方式，用户可以通过 Chrome Web Store 直接安装，且拥有自动更新功能。

### 准备工作
1. 确保你有一个 Google 账号。
2. 准备一张支持国际支付的信用卡（需要支付 $5 一次性注册费）。
3. 准备好插件的图标（128x128）、截图（至少一张 1280x800 或 640x400）和宣传图（440x280）。

### 步骤
1. **注册开发者账号**：
   访问 [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/developer/dashboard)，登录并支付注册费。

2. **上传插件**：
   - 点击 "Create new item"（创建新项目）。
   - 上传本项目根目录下生成的 `release.zip` 文件（如果没有，请先运行 `zip -r release.zip dist` 或手动压缩 `dist` 文件夹）。

3. **填写信息**：
   - **Store Listing**：填写名称、描述、类别（建议选 Developer Tools）、语言等。
   - **Privacy**：填写隐私政策（由于本插件只使用本地存储 `storage`，不收集用户数据，可以在描述中声明）。
     - 在 "Permissions" 栏目中，解释为什么需要 `storage` 权限（用于保存用户偏好设置）。

4. **提交审核**：
   - 完成所有必填项后，点击 "Submit for review"。
   - 审核通常需要 1-3 个工作日。

## 方式二：手动分发源代码（加载已解压）

如果你不想支付注册费，或者只是在小范围内分享（如公司内部、朋友间）。

### 步骤
1. 将 `release.zip` 发送给使用者。
2. 使用者解压该文件。
3. 打开 Chrome 浏览器，访问 `chrome://extensions/`。
4. 打开右上角的 **"开发者模式" (Developer mode)** 开关。
5. 点击左上角的 **"加载已解压的扩展程序" (Load unpacked)**。
6. 选择解压后的文件夹（即 `dist` 文件夹的内容）。

> **注意**：Chrome 为了安全，可能会在每次重启浏览器后提示“停用以开发者模式运行的扩展程序”，用户需要手动点击取消。

## 方式三：使用 Chrome 自带打包 (.crx)

Chrome 提供了生成 `.crx` 安装包的功能，但**现代 Chrome 版本出于安全考虑，不再允许直接拖拽安装非商店的 .crx 文件**（除非是企业策略分发）。这种方式主要用于自签名分发给其他基于 Chromium 的浏览器（如 Edge, Brave 等可能策略不同）或用于企业内部部署。

### 步骤
1. 打开 Chrome 扩展管理页面：`chrome://extensions/`。
2. 开启右上角的 **"开发者模式"**。
3. 点击顶部的 **"打包扩展程序" (Pack extension)** 按钮。
4. 在弹出的对话框中：
   - **扩展程序根目录**：选择本项目的 `dist` 目录（**注意：不是项目根目录，是构建后的 `dist` 目录**）。
   - **私钥文件**：首次打包留空（Chrome 会自动生成）；如果是更新版本，需要选择之前生成的 `.pem` 文件。
5. 点击 **"打包扩展程序"**。

### 产物
Chrome 会在 `dist` 目录的**同级目录**（即项目根目录）下生成两个文件：
- `dist.crx`：插件安装包。
- `dist.pem`：私钥文件（**非常重要，请妥善保管**）。后续更新插件版本时，必须使用同一个 `.pem` 文件打包，否则 ID 会变，无法覆盖旧版本。

### 如何安装 .crx
由于 Chrome 限制，普通用户无法直接双击或拖拽安装 `.crx`。安装方法通常是：
1. 打开 `chrome://extensions/`。
2. 开启 **"开发者模式"**。
3. 将 `.crx` 文件拖入该页面。
*(注：如果 Chrome 提示“该扩展程序未列在 Chrome 应用商店中，并可能是在您不知情的情况下添加的”，则会被强制禁用。此时只能用方式二)*
